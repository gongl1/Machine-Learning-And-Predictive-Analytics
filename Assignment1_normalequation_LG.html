# Step 4: Calculate metrics for the test data

# Mean squared error
mse = np.mean((y_pred - y_test) ** 2)

# R-squared
ss_res = np.sum((y_test - y_pred) ** 2)
ss_tot = np.sum((y_test - np.mean(y_test)) ** 2)
r_squared = 1 - (ss_res / ss_tot)

# Explained variance.
# Explained variation measures the proportion to which a mathematical model accounts for the variation of a given data set.
explained_variance = 1 - (np.var(y_test - y_pred) / np.var(y_test))

# Print metrics
print("Mean Squared Error:", np.round(mse, 2))
print("R-squared:", np.round(r_squared, 2))

Part E: Conceptual Questions¶

1. Why is it important to have a test set¶

The test set simulates real-world data that the model is likely to encounter in production. By assessing the model's performance on previously unseen examples, we can gauge its ability to generalize and make accurate predictions on new, unseen data. The test error from test set is the generalization error.

2. If the normal equation always provides a solution, when would we not want to use it?¶

While the normal equation provides an analytical solution to linear regression, there are situations where it may not be the preferred method:

  1. The normal equation involves computing the inverse of a matrix. For large datasets with a high number of features, computing the inverse may be computationally expensive and memory-intensive.

  2. Numerical Stability: Inverting a matrix can lead to numerical instability, especially if the matrix is ill-conditioned or singular (non-invertible). This can result in numerical errors and inaccuracies in the computed coefficients.

  3. Memory Constraints: Inverting a large matrix requires storing the entire dataset in memory, which may not be feasible for datasets that exceed available memory resources.

  4. Regularization: The normal equation does not directly support regularization techniques like Ridge Regression or Lasso Regression, which are used to prevent overfitting by penalizing large coefficients. In such cases, iterative optimization algorithms like Gradient Descent or stochastic gradient descent (SGD) with regularization may be more suitable.

  5. Online Learning: In scenarios where data arrives sequentially or in batches (e.g., streaming data), the normal equation is not applicable as it requires access to the entire dataset at once. Online learning algorithms that update the model parameters incrementally are more appropriate in such cases.

  6. Feature Scaling: The normal equation may perform poorly if features have vastly different scales. In such cases, feature scaling (e.g., normalization or standardization) is required to ensure numerical stability and convergence of the normal equation.

4. As we move further into Machine Learning, we will need to continually consider the bias-variance tradeoff. Explain what bias is and what variance is in regards to the bias-variance tradeoff.¶

Mathematically

bias = training set error

variance = (train - test) set error

Bias: Bias refers to the error introduced by approximating a real-world problem with a simplified model. It represents the difference between the average prediction of the model and the true value we are trying to predict. A high bias model tends to underfit the data, meaning it oversimplifies the underlying relationships between variables and performs poorly on both the training and test datasets.

Variance: Variance measures the model's sensitivity to small fluctuations or noise in the training data. It represents the amount by which the model's predictions would change if trained on different subsets of the data. A high variance model tends to overfit the training data, capturing noise in the data rather than the underlying patterns, and performs well on the training dataset but poorly on the test dataset.

In brief, bias is the error introduced by the simplifying assumptions of the model, while variance is the error introduced by the model's sensitivity to noise in the training data. The bias-variance tradeoff refers to the delicate balance between these two sources of error: reducing bias typically increases variance and vice versa. The goal is to find the optimal balance that minimizes the model's overall error on unseen data.

5. In a linear regression model, how might we reduce bias?¶

Reducing bias in a linear regression model involves increasing model complexity, incorporating more relevant features, and relaxing assumptions to better capture the underlying patterns in the data.

6. In a linear regression model, how might we reduce variance?¶

Reducing variance in a linear regression model involves techniques that aim to prevent overfitting to the training data and encourage generalization to unseen data.

Regularization: Regularization techniques like Ridge Regression and Lasso Regression add a penalty term to the loss function, which discourages large coefficients. This helps reduce variance by preventing the model from fitting the noise in the training data too closely.

Feature Selection: Selecting only the most relevant features and discarding irrelevant or redundant ones can reduce variance by simplifying the model and preventing overfitting to noisy or irrelevant features.

Cross-Validation: Using techniques like k-fold cross-validation allows the model to be trained and evaluated on multiple subsets of the data. This helps estimate the model's performance more accurately and reduces the likelihood of overfitting to a specific training set.

Ensemble Methods: Ensemble methods like Random Forests and Gradient Boosting combine multiple weak learners (e.g., decision trees) to create a more robust and generalizable model. By averaging predictions from multiple models, ensemble methods can reduce variance and improve performance.

Early Stopping: In iterative optimization algorithms like Gradient Descent, monitoring the model's performance on a validation set and stopping training when performance starts to degrade can prevent overfitting and reduce variance.

Simplifying the Model: Using simpler models with fewer parameters can reduce variance by limiting the model's capacity to fit the training data too closely. For example, using linear regression instead of polynomial regression can reduce variance at the cost of potentially increased bias.

Increasing Training Data: Increasing the size of the training dataset can help the model generalize better to unseen data by providing more diverse examples for training. This can reduce variance by exposing the model to a wider range of scenarios and reducing the impact of noise in the training data.

The Normal Equation provides an analytical solution to the linear regression problem by directly computing the coefficients based on the training data. While it's a straightforward approach for small datasets, it can be computationally expensive and numerically unstable for large datasets or datasets with multicollinearity.

In contrast, scikit-learn's LinearRegression class typically employs optimization algorithms that are more efficient and scalable for larger datasets. These algorithms iteratively minimize a cost function, such as the mean squared error, to find the optimal coefficients. Some common optimization algorithms used by scikit-learn's LinearRegression class include Ordinary Least Squares (OLS), Gradient Descent, and its variants (e.g., Stochastic Gradient Descent, Mini-batch Gradient Descent).
